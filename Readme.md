### SRP (Single Responsibility Principle)

Une classe ne doit avoir qu'une seule raison de changer. Cela améliore la maintenabilité et réduit le couplage.

### OCP (Open-Closed Principle)

Une classe doit être ouverte à l'extension mais fermée à la modification. On utilise l'abstraction et le polymorphisme.

### LSP (Liskov Substitution Principle)

Les objets d'une classe dérivée doivent pouvoir remplacer les objets de la classe de base sans altérer le comportement.

### ISP (Interface Segregation Principle)

Il vaut mieux avoir plusieurs interfaces spécifiques qu'une seule interface générique. Évite les dépendances inutiles.

### DIP (Dependency Inversion Principle)

Il faut dépendre des abstractions, pas des implémentations concrètes. Cela favorise la flexibilité et la testabilité.
